# Open Design：题库网站优化方案

## 参考方向

- VitePress / Nextra / Astro Starlight：内容优先、导航清晰、正文阅读体验稳定。
- Quartz：面向 Obsidian/Markdown 知识库，适合把笔记发布成长期维护的网站。
- MathJax + markdown-it：前者负责 TeX 公式渲染，后者负责浏览器端 Markdown 阅读。

## 设计目标

- 行动优先：打开页面后先看到今日路线、下一道该处理的错题和关键健康状态。
- 扫描清楚：把总量、进度、待核对、系统问题分成稳定的信息层级，减少同权重卡片堆叠。
- 数据开放：所有入口仍指向本地 Markdown、看板和 LLM Wiki，不把来源藏在页面里。
- 复习友好：保持工具型页面的密度，但给关键区域留出呼吸感，适合每天重复打开。
- 公式可靠：站内 Markdown 链接进入 `reader.html`，由 MathJax 渲染 `\(...\)`、`\[...\]`、`$...$` 和 `$$...$$`。

## 页面结构

- 顶部：品牌、常用入口和一张轻量学习桌面横幅，建立“数学复习工作台”的第一印象。
- 今日焦点：展示学习闭环中的第一项，包含错题、状态、掌握度和防错提示。
- 指标条：保留核心数字，但按“今日队列、二轮、五年、AI、Wiki、系统”组织。
- 工作区：今日学习闭环、题库进度、系统健康、高频信号、本次维护。

## 视觉规范

- 风格：参考优秀开源文档站，内容优先，少装饰，少大卡片，减少“AI 生成感”。
- 主色：蓝色用于主要行动和链接，绿色只用于进度/完成状态，琥珀/红色只用于提醒。
- 圆角：控制在 8px 以内，保持工具界面而不是营销卡片感。
- 字号：首页标题适度突出；大数字只用于指标；面板内标题保持紧凑，避免页面被标题撑大。
- 质感：弱化阴影和装饰图，使用细线、留白和排版建立品质感。
- 移动端：今日路线转成纵向信息块，其余表格保持可横向查看。
