# Claude Code / DeepSeek v4 Pro 整理说明

当你在 Claude Code 中使用 DeepSeek v4 Pro 时，可以让它按这份说明整理 `08_AI笔记管理/01_待整理` 里的 md。当前推荐流程是一节课或一个题组导出一个 AI 对话文件，再整理成一份标准笔记。

## 工作方式

1. 一份原始 md 原则上整理成一份标准笔记；一节 AI 对话整理成一份本节主笔记。
2. 输出位置放到 `08_AI笔记管理/02_标准笔记`，尽量按题库专题和讲次分文件夹。
3. 标准笔记必须符合 `08_AI笔记管理/90_模板/标准笔记模板.md`。
4. 分类要优先匹配 `04_原始资料/problems.json` 中已有的 `module_id`、`module`、`lesson_id`、`topic`。
5. 原文里出现题号时，尽量写入 `related_problem_ids`，并在“例题关联”中说明。
6. 原文信息不足时，不要补编；把 `status` 写成 `review`，把问题写进“待核对”。
7. 公式用 Obsidian 可识别格式：行内 `\( ... \)`，独立公式 `\[ ... \]`。
8. AI 对话不是标准答案；关键结论、题号、图片信息不能核对时，写入“待核对”。
9. 如果对话里值得拆出错题或方法卡，先在“待核对”里列拆分建议。

## 标准正文小节

正文必须包含并保持这个顺序：

```markdown
## 原始问题

## 一句话结论

## 知识归属

## 核心内容

## 方法流程

## 例题关联

## 易错点

## 下次复习

## 待核对
```

## 推荐给 Claude Code 的一句话

请读取 `08_AI笔记管理/01_待整理/00_AI对话导入入口` 中尚未整理的 md/txt 文件，使用 DeepSeek v4 Pro 按 `08_AI笔记管理/90_模板/标准笔记模板.md` 和 `08_AI笔记管理/90_模板/AI对话整理要求.md` 整理成标准笔记，写入 `08_AI笔记管理/02_标准笔记`，最后运行 `python tools/ai_note_manager.py index` 刷新索引。
